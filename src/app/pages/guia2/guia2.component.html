Guía: Cifrado Asimétrico usando LUC

1. Conceptos clave del criptosistema LUC

El sistema de cifrado LUC, también conocido como Lucas-based Public Key Cryptosystem, utiliza secuencias de Lucas, que son similares a la secuencia de Fibonacci pero con propiedades matemáticas más generales. La fórmula principal utilizada es:

Fórmula de la secuencia de Lucas:

Un(P, Q) = P * Un-1(P, Q) - Q * Un-2(P, Q)

Donde:

  • Un(P, Q): Es el número de Lucas en la posición n, basado en los parámetros P y Q.
  • P y Q: Son enteros elegidos al inicio.
  • U0 = 0 y U1 = 1 como condiciones iniciales.

2. Generación de las claves pública y privada

En el criptosistema LUC, como en otros métodos asimétricos, se generan dos claves: una pública para cifrar y una privada para descifrar.

  • Paso 1: Elige dos números primos grandes, p y q.
  • Paso 2: Calcula N = p * q. Este valor se usará para el cifrado y descifrado.
  • Paso 3: Selecciona los parámetros P y Q, generalmente se usa Q = 1.
  • Paso 4: Elige el exponente de cifrado e tal que sea coprimo con (p-1) * (q-1).
  • Paso 5: Calcula la clave privada d, de modo que e * d sea congruente a 1 módulo lambda(N), donde lambda(N) es el mínimo común múltiplo de p-1 y q-1.

3. Proceso de cifrado

Para cifrar un mensaje M (convertido en un número), se utiliza la secuencia de Lucas y la clave pública e:

Fórmula de cifrado:

C = Ue(P, Q) mod N

Donde Ue(P, Q) es el término de la secuencia de Lucas en la posición e y C es el mensaje cifrado.

4. Proceso de descifrado

El descifrado se realiza usando la clave privada d para recuperar el mensaje original M:

Fórmula de descifrado:

M = Ud(P, Q) mod N

Aquí Ud(P, Q) es el término de la secuencia de Lucas en la posición d, lo que permite obtener el mensaje original.

5. Ejemplo básico de cifrado y descifrado

A continuación se muestra un ejemplo simplificado:

  • Generación de claves:
    • Primos: p = 11, q = 19.
    • Cálculo: N = p * q = 209.
    • Parámetros: P = 3, Q = 1.
    • Clave pública e = 7, clave privada d = 13.
  • Cifrado: Para un mensaje M = 5, calcula C = U7(3, 1) mod 209.
  • Descifrado: Con la clave privada d = 13, se calcula M = U13(3, 1) mod 209.

6. Ventajas y desventajas del criptosistema LUC

Ventajas:

  • Resistencia a ciertos ataques: Es más robusto frente a ciertos ataques algebraicos que otros sistemas como RSA.
  • Diversidad: Al usar una secuencia diferente (Lucas), proporciona mayor variabilidad matemática.

Desventajas:

  • Complejidad computacional: Es más complicado de implementar debido a los cálculos de la secuencia de Lucas.
  • Rendimiento: Generalmente es más lento en comparación con RSA.

7. Aplicaciones del sistema LUC

LUC puede ser utilizado en escenarios donde se necesite un alto grado de seguridad y resistencia a ataques, como en:

  • Sistemas de firma digital.
  • Cifrado de mensajes en entornos altamente seguros.
  • Sistemas financieros con alta demanda criptográfica.

8. Conclusión

El criptosistema LUC es una alternativa interesante a otros métodos de cifrado asimétrico. Su uso de la secuencia de Lucas proporciona una robustez adicional frente a ciertos ataques, aunque con una mayor complejidad computacional.
